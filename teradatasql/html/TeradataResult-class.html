TeradataResult class

Description

TeradataResult class

Bind values to prepared statement parameter markers dbBind(res, params, ...)

Close the result dbClearResult(res, ...)

Obtain result column metadata dbColumnInfo(res, ...)

Fetch rows dbFetch(res, n = -1, ...) An attempt to fetch from a closed result set raises an error.

Obtain the number of rows fetched dbGetRowCount(res, ...)

Obtain the number of rows affected by the SQL statement dbGetRowsAffected(res, ...)

Obtain the SQL request text dbGetStatement(res, ...)

Determine whether the entire result has been fetched dbHasCompleted(res, ...)

Determine whether the result is usable dbIsValid(dbObj, ...)

Advances to the next result set. dbNextResult(res, ...)

Sets the fetched row count for testing. setFetchedRowCount(res, ...)

Arguments

res

A TeradataResult object

params

Parameter values to bind

n

Maximum number of rows to fetch. Specify n = -1 or n = Inf to fetch all remaining rows. If the n argument is not an atomic whole number greater or equal to -1 or Inf, an error is raised.

dbObj

A TeradataResult object

u

The fetched row count

Value

The TeradataResult invisibly

TRUE invisibly

A data frame with columns name (type character), Sclass (R data type), type (type character), len (type integer), precision (type integer), scale (type integer), nullOK (type logical). The data frame contains one row for each result column.

A data frame with columns and rows corresponding to result set columns and rows. If the SQL statement returned an empty result set, the data frame will have fully-typed columns but zero rows. If the SQL statement did not return a result set, the data frame will have zero columns and zero rows. The column types of the returned data frame depend on the data returned: integer for 32-bit integer values bit64::as.integer64() for 64-bit integer values numeric for numbers with a fractional component character for text raw for blobs as.Date() for dates hms::as_hms() for times as.POSIXct() for timestamps NA for SQL NULL values
